# ADR — Next.js BFF vs direkt API-call a kgc-berles → KGC-4 között

> **Status:** draft-research — a [[06-Audits/2026-05-18 KGC-4 integráció — architektúra v1|architektúra v1]] alapján Peti finomítja.

## Kérdés

A Next.js front a KGC-4 portal-API-ját **közvetlenül a browser-ből** vagy **BFF-route-on (Next.js `/api/*`)** keresztül érje el?

## Opciók

- (A) **BFF-route mindenre** (`/api/equipment`, `/api/booking`, `/api/revalidate`) — CORS-mentes, token-rejtés, response-shape transformation
- (B) **Direkt API-call a browser-ből** — egyszerűbb, de CORS + tenant-ID exponálás
- (C) **Hybrid:** anonim read = direkt (gyorsabb), auth-mutation = BFF (token-protect)

## Tradeoff-mátrix összefoglaló

Teszt-eu/Balance projektben már bevált a BFF-bridge pattern: [[11-wiki/nextjs-api-proxy-bridge]]. Performance vs maintainability. Részletek: [[06-Audits/2026-05-18 KGC-4 integráció — architektúra v1]]

## Várt döntés

Egy konkrét stratégia + példa-kódminták a Next.js BFF-route-okra.
