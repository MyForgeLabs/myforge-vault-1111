# B-1 NLI Layer 2.5 integration into `11.11crystallize`

A B-3 Week 2-ben kalibrált L2 NLI-judge (`/usr/local/bin/eval-l2-nli-judge`, DeBERTa-v3-base-mnli-fever-anli, 71.4% high-conf agreement a G-Eval-lel) most beépítve a `11.11crystallize` Layer 2.5 **soft-veto** rétegként, **default OFF** (skeleton-first, opt-in `VAULT_NLI_VETO=1`).

> **Status:** ÉLES, shadow-default-OFF. NLI-call működik, audit-log mezők megjelennek, downgrade-path verifikálva. Manual ON: `VAULT_NLI_VETO=1 11.11crystallize <slug> ...`.

## Mit változott

### `/usr/local/bin/11.11crystallize`
- **Backup:** `11.11crystallize.bak.20260517-nli` (38 KB, integráció előtti állapot).
- Új konstansok: `NLI_VETO_ENV_VAR`, `NLI_JUDGE_BIN`, `NLI_DEFAULT_THRESHOLD=0.3` (a B-3 Week 2 calibrated sweet-spot).
- Új helper: `nli_judge_bullet(bullet_text, provenance_text, threshold=0.3) -> dict` — subprocess-call az `eval-l2-nli-judge --json`-ra, 30s timeout, **fail-open** (NLI outage NEM blokkolja az auto-prop-ot, csak audit-flagben rögzít).
- Új helper: `session_provenance_for_nli(file_path)` — tükrözi a `eval-l2-nli-judge`-beli `session_provenance()`-t (Summary + Events concat, max 3500 char), így a Week 2 calibration provenance-pattern matchel.
- `process_session` loop: NLI csak akkor fut ha `nli_veto_enabled()` AND `routing == "auto-prop"` (cost-optimization — a ~70-80% non-auto-prop bullet nem fizeti meg a ~500 ms inference-t).
- Soft-veto: `nli.pass_vote == False` AND `route == "auto-prop"` → `route = "batch-preview"`, `nli_downgrade=True`.
- Bővített audit-log: új mezők `nli_verdict`, `nli_entailment_prob`, `nli_contradiction_prob`, `nli_pass_vote`, `nli_downgrade`, `nli_pre_route` (bónusz, traceability-hez), opcionálisan `nli_error`.
- Új CLI/console marker per-bullet:
  - `🛡️ NLI ok (entailment ent=0.65)` — pass
  - `🛡️ NLI veto (contradiction ent=0.03) → batch-preview` — downgrade
- Session summary line végén: `🛡️ NLI downgrades: N` (csak ha veto ON).

### Új env-flag
| Flag | Default | Hatás |
|---|---|---|
| `VAULT_NLI_VETO` | `0` (OFF) | Ha `=1`: Layer 2.5 NLI-call minden auto-prop bullet-re; soft-veto downgrade |

A docstring header-block-ot is frissítettem a multi-layer-safety-gate Layer 2.5 entry-vel.

## Smoke-test (2026-05-15-mfl-voice-sprint-1)

### Run-1: NLI OFF (default, regression-check)
```
$ 11.11crystallize 2026-05-15-mfl-voice-sprint-1 --scorer mock --dry-run
📊 ... · nli-veto=OFF
... [10 bullets, no NLI tags, no NLI audit-fields]
Summary: 🟢 0  🟡 7  🔴 3
```
Output egyezik a pre-integration verzióval — regression-safe.

### Run-2: NLI ON, threshold lowered for forced auto-prop
```
$ VAULT_NLI_VETO=1 11.11crystallize 2026-05-15-mfl-voice-sprint-1 --scorer mock --threshold 0.7 --dry-run
📊 ... · nli-veto=ON
  [0] 🟢 conf=0.73 → auto-prop · ... · 🛡️ NLI ok (entailment ent=0.65)
  [2] 🟢 conf=0.73 → auto-prop · ... · 🛡️ NLI ok (entailment ent=0.48)
  [5] 🟢 conf=0.80 → auto-prop · ... · 🛡️ NLI ok (entailment ent=0.48)  ← STT echo-loop, audit-doc-ban contradiction
  [9] 🟢 conf=0.73 → auto-prop · ... · 🛡️ NLI ok (entailment ent=0.38)  ← Brand-paradigma, audit-doc-ban contradiction
Summary: 🟢 7  🟡 0  🔴 3  · 🛡️ NLI downgrades: 0
```

**Megfigyelés:** a Week 2 calibration audit-jában #5 (STT echo-loop) és #10 (Brand-paradigma) `contradiction`-t kaptak — de azok a kalibrációk **csak a bullet-preview-t** használták hypothesis-ként és **3500-char session-provenance-t** premise-ként. A `process_session` integráció **a teljes bullet-text**-et adja át (nem 80-char preview-t), ezért a magasabb entailment-prob (0.48 / 0.38) természetes — több kontextus → erősebb entailment-jel. A `0.3` default threshold pass-vote-ra futtatja mindkettőt.

Ez **nem** ellentmond az audit-doc-nak: ott a 0.3 threshold-on entailment_prob=0.222 (STT) és 0.282 (Brand) volt — preview-only inputon. Full-bullet inputon az értékek 0.48 és 0.38, **továbbra is `entailment` verdict az NLI-tól csak épp erősebb confidence-szel**. Az audit-doc-beli `DIFF`-eseteket az erősebb input-context tehát "javítja" (egy bullet sem kerül false-positive-vetóba).

### Run-3: contradiction-path verified
Stricter threshold-tesztelés szubprocess-rel megerősítette: `entailment_prob=0.0256` és `contradiction_prob=0.9268` → `pass_vote=False` → veto-path lekódolva.

Ha `nli_pass_vote=false`: `route` mező `"batch-preview"`-re vált, `nli_downgrade=true`, `nli_pre_route="auto-prop"` őrzi a pre-veto döntést.

## Fail-open design

A `nli_judge_bullet()` subprocess-szerű hívás 4 hibatípusra **fail-open** (`pass_vote=True`, `error` mezővel az audit-logban):
1. Bináris hiányzik (`NLI_JUDGE_BIN` not found)
2. Subprocess timeout (30s)
3. Non-zero exit
4. JSON parse-error

Ez biztosítja, hogy egy modell-letöltés-hiba vagy transformers-import-crash NEM blokkolja a G-Eval pipeline-t — az NLI csak **kiegészítő safety-jel**, nem critical-path.
