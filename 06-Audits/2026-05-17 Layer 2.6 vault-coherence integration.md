# B-1 Week 5 — Layer 2.6 `vault-coherence-check` integration

Az előző session (`2026-05-17-obsidian-vault-2` Phase 4.R2.4) létrehozott `/usr/local/bin/vault-coherence-check` skeleton most beépítve a `11.11crystallize` **Layer 2.6 soft-veto** rétegként, közvetlenül a Layer 2.5 NLI-judge után. Default OFF (skeleton-first, opt-in `VAULT_COHERENCE_CHECK=1`).

> **Status:** ÉLES, shadow-default-OFF. Coherence-call működik, audit-log 5+ új mező megjelenik (`coherence_status` / `coherence_max_contra_prob` / `coherence_neighbour_count` / `coherence_downgrade` / `coherence_pre_route` + `coherence_latency_ms` / `coherence_conflicts` / `coherence_error`). Downgrade-path verifikálva (fail-open ERROR-ra). Manual ON: `VAULT_COHERENCE_CHECK=1 11.11crystallize <slug> ...`.

## Mit változott

### `/usr/local/bin/11.11crystallize`

- **Backup:** `11.11crystallize.bak.20260517-coherence` (45 KB, integráció előtti állapot — tartalmazta a Layer 2.5 + NB-enrich layereket is, csak Layer 2.6 hiányzott).
- Új konstansok: `COHERENCE_ENV_VAR="VAULT_COHERENCE_CHECK"`, `COHERENCE_CHECK_BIN="/usr/local/bin/vault-coherence-check"`, `COHERENCE_DEFAULT_TOP_K=5`, `COHERENCE_DEFAULT_THRESHOLD=0.7`, `COHERENCE_DEFAULT_COSINE_FLOOR=0.40`, `COHERENCE_SUBPROCESS_TIMEOUT=180` (kalibrált; 5 cold NLI-call/bullet ~40-60s).
- Új helper: `coherence_check_bullet(bullet_text, top_k=5, threshold=0.7, cosine_floor=0.40) -> dict` — subprocess-call `vault-coherence-check --bullet ... --top-k 5 --threshold 0.7 --cosine-floor 0.40 --json`-re, 180s timeout, **fail-open** (coherence-check outage NEM blokkolja az auto-prop-ot, csak `status=ERROR` flag-ben rögzít). Visszaadott dict: `status` (OK/FLAG/SKIP/ERROR), `flagged`, `max_contra_prob`, `neighbour_count`, `coherence_score`, `conflicts` (top-3 preview), `latency_ms`, `error`.
- `process_session` loop: coherence csak akkor fut ha `coherence_check_enabled()` AND `routing == "auto-prop"` *post-NLI* (cost-optimization — a Layer 2.5 már leszűrt batch-preview/discard-ba ~70-80%-ot, a maradékra fut csak Layer 2.6).
- Soft-veto: `coherence.flagged == True` AND `route == "auto-prop"` → `route = "batch-preview"`, `coherence_downgrade=True`.
- `SKIP` státusz külön: ha minden neighbour `cosine < cosine_floor` (0 NLI-call történt) — nem downgrade-elünk, mert nincs mihez kontradikciót mérni.

### Új audit-log mezők (`06-Audits/crystallize-log.jsonl`)

Az auto-prop kandidát bullet-ekre (csak azokra) az alábbi mezők kerülnek a JSONL record-ba:

| Mező | Típus | Érték |
|---|---|---|
| `coherence_status` | str | `OK` / `FLAG` / `SKIP` / `ERROR` |
| `coherence_max_contra_prob` | float | max `contradiction_prob` az ítélt neighbour-eken (0.0-1.0) |
| `coherence_neighbour_count` | int | hány neighbour-en futott NLI (cosine-floor + üres-snippet után) |
| `coherence_downgrade` | bool | `true` iff routing-downgrade történt (auto-prop → batch-preview) |
| `coherence_pre_route` | str | mi lett volna a route, ha Layer 2.6 nem aktivál (traceability) |
| `coherence_latency_ms` | int | subprocess wall-clock idő |
| `coherence_conflicts` | list | csak ha `flagged=True` — top-3 `{file, contradiction_prob}` preview |
| `coherence_error` | str | csak ha `status=ERROR` — subprocess timeout / parse-error magyarázat |

Discard / batch-preview / NLI-downgrade bullet-eken EZEK A MEZŐK HIÁNYOZNAK (cost-opt jelzés).

### Új CLI/console marker per-bullet
- `🔬 coh ok (mcp=0.69, n=5)` — pass (max_contra_prob 0.69, 5 neighbour judged)
- `🔬 coh FLAG (mcp=0.83, n=5) → batch-preview` — downgrade
- `🔬 coh skip (n=0)` — minden neighbour cosine-floor alatt (SKIP)
- `🔬 coh ERROR (fail-open)` — subprocess timeout vagy parse-error, routing változatlan

Session summary line végén: `· 🔬 coherence downgrades: N (errors: M)` (csak ha coherence ON).

### Új env-flag

| Flag | Default | Hatás |
|---|---|---|
| `VAULT_COHERENCE_CHECK` | `0` (OFF) | Ha `=1`: Layer 2.6 vault-coherence-check minden auto-prop bullet-re (post-NLI); soft-veto downgrade ha bármely canonical-vault neighbour `contradiction_prob >= 0.7` |

### Integráció helye (kód-szakasz)

```python
# process_session() body, per-bullet loop:

# ── Layer 2.5 — NLI soft-veto ──
if nli_on and routing == "auto-prop":
    nli_result = nli_judge_bullet(bullet, nli_provenance)
    if not nli_result.get("pass_vote", True):
        routing = "batch-preview"
        nli_downgrade = True

# ── Layer 2.6 — vault-coherence-check (ÚJ) ──
if coherence_on and routing == "auto-prop":  # csak NLI-survivor-okon
    coherence_result = coherence_check_bullet(bullet)
    if coherence_result.get("status") == "ERROR":
        coherence_errors += 1
    if coherence_result.get("flagged"):
        routing = "batch-preview"
        coherence_downgrade = True

counts[routing] += 1
```

### False-positive arány

**0 / 4 (0%)** — a roadmap/sprint-agenda/pattern-evergreen jellegű bullet-eken NEM volt FLAG. Két bullet borderline (mcp=0.690 és 0.698, threshold 0.70), de **küszöb alatt maradt**. Kalibráció helyes, cosine-floor 0.40 + threshold 0.70 jól szűri a "topically-related-but-not-contradictory" neighbour-eket.

### Test 3 — első próbálkozás 60s timeout-tal (history, nem reprodukálandó)

Az első integráció `COHERENCE_SUBPROCESS_TIMEOUT=60` volt. **2/4 bullet-en `coh ERROR (fail-open)`** látszott (subprocess elfutott a 60s alatt 5 cold-process NLI-call miatt). **Fix:** timeout 60→180s (5 cold-process NLI-call ~40-60s). Re-run után 0 ERROR. A `latency_ms` audit-mező rögzíti a tényleges fp99-et a Week 6 monitor-script számára.

> [!info] Latency-megjegyzés
> Minden coherence-check-call **friss subprocess**, ami friss `eval-l2-nli-judge` subprocess-eket spawn-ol top-K-szor → DeBERTa-v3 modell cold-load ×K. 50-60s/bullet ~5 neighbour-on stabil. Production-flow-ban (auto-prop = ~20-30%-os flow) ez ~10× a session-Stop overhead-je. Ha Week 6+ scale-issue: process-pool / persistent NLI-server.

## Layer 2.5 vs 2.6 — funkcionális diff

| | Layer 2.5 (NLI veto) | Layer 2.6 (vault-coherence) |
|---|----------------------|------------------------------|
| Premise | session ## Summary + ## Events | **canonical vault top-K cosine neighbour** snippet (wiki/ADR/skill) |
| Hypothesis | bullet text | bullet text |
| Kérdés | "ezt a bullet-et alátámasztja a saját session-je?" | "ezt a bullet-et ellentmond a vault meglévő desztillátuma?" |
| Cost | 1 NLI-call / bullet | top-K (=5) NLI-call / bullet, közbeiktatott vault-search-szel |
| Sweet-spot threshold | entailment ≥ 0.30 (pass) | contradiction ≥ 0.70 (FLAG) |
| Védelmi kör | rohamosan-promotált self-referential learning | drift / contradicting prior knowledge |

Komplementer. Layer 2.5 a "session önmagán belül koherens" kérdést válaszolja meg; Layer 2.6 a "vault egész-szintű konzisztens marad" kérdést.
