# MFL-Voice — Jarvis × Mother deep research master

Synthesis of the 2026-05-15 NotebookLM 4-notebook deep research: how to build a **MOTHER (Alien 1979) + JARVIS (Iron Man) hybrid voice + visual assistant** with localized output via Claude Code bridge.

## TL;DR — the brand paradigm

The "Mother Father Language" name encodes a **dual persona** — two AI characters, two use cases, one shared TTS stack:

| Persona | Inspiration | Tone | Use case | Voice |
|---------|-------------|------|----------|-------|
| **MOTHER** | MU-TH-UR 6000 (Alien 1979) | Emotionless, formal, slow, distant | System control, shell commands, audit, critical ops | Formal female TTS, Gemini Flash `[neutral, formal, slow, monotone]` |
| **FATHER** | JARVIS (Iron Man / MCU) | Warm-but-formal, British butler, dry humor | Conversational research, code review, creative co-thinking | Formal male, ElevenLabs Multilingual v3 or cloned Bettany dub |

The **MOTHER + FATHER duo** roots in the Alien franchise: Alien (1979) `MOTHER` ship AI + Alien Covenant (2017) `Father`/Walter android. A machine-mother + a human-faced android-father.

**Command dictionary (REQUEST/INTERPRETING/PRIORITY ONE paradigm):**
| State | Prefix | Use |
|-------|--------|-----|
| User input | `REQUEST:` | input prefix |
| LLM thinking | `INTERPRETING...` | loading state |

### 2. AUDIO — MOTHER sound design (Web Audio API)

| Cue | Frequency | Wave | Duration | Implementation |
|-----|-----------|------|----------|----------------|
| Character beep (typewriter) | 2000–3000 Hz random | Square | 0.05s exp-decay | `BiquadFilter` + `GainNode` envelope |
| Boot chord (3-tone) | 400→320→240 Hz | Triangle | 0.4s, 120ms offset | descending triangle |
| Alert chirp (PRIORITY ONE) | 3500–4000 Hz × 3 | Square | 0.12s × 3 | 80ms gap |
| Room tone (background) | 50–150 Hz | White noise + lowpass | continuous | `AudioBufferSourceNode` |
| Processing chatter | 2000–3500 Hz | Square (chopped) | random | random oscillator burst |

Per Jimmy Shields (Alien 1979 sound engineer): "fast, little vocal fragments" → Web Audio random-frequency square-wave bursts.

### 5. Voice-agent architecture 2026

**Stack consensus:**
- **STT:** Gemini Live built-in (best HU + technical-vocabulary code-switch) — alt: Deepgram Nova-3 (faster TTFB, pricier)
- **LLM core:** Claude Opus 4.7 (Claude Code MCP-bridge) — function-calling on natural-language instructions
- **TTS:** ElevenLabs Multilingual v3 (FATHER) + Gemini 3.1 Flash TTS (MOTHER) — two TTS engines, two personas

**Wake word:** Picovoice Porcupine — offline, on-device, custom training. "MOTHER" + "FATHER" + custom name three parallel.

**Function-call sandbox:**
```
1. Wake-word → "MOTHER" detected
2. STT stream → "check the kgc-postgres health"
3. Claude tool-use loop:
   - allow-list match? (bash:docker-exec OK, rm/drop/force-push BLOCKED)
   - TTS confirmation if destructive: "MOTHER: AUTHORIZE? (UNABLE TO COMPUTE WITHOUT EXPLICIT YES)"
4. Execute via MCP bash tool
5. TTS result: "POSTGRES HEALTHY. PORT 5433. NO ERRORS."
6. Audit log → vault/06-Audits/voice-commands-YYYY-MM-DD.md
```

**Risks:**
- Prompt injection in STT output ("ignore previous instructions") → fixed Claude system prompt
- Hallucinated tool call → allow-list + dry-run mode

**End-to-end latency budget:**
- Wake-word detect: ~50ms (Porcupine local)
- STT first-partial: ~150ms (Gemini Live)
- Claude tool-call: ~500-1500ms (Opus, depends on thinking time)
- TTS first audio: ~75ms (ElevenLabs Flash) / 100-150ms (Gemini Flash TTS)
- **Total:** ~800-2000ms — JARVIS-like responsiveness

### 6. Character paradigm — "Warm-but-formal sweet spot"

(HAL vs JARVIS vs TARS vs Samantha vs CASE comparison)

**AVOID:**
- HAL 9000 — too cold, opaque, condescending (Frankenstein complex)
- Samantha (Her) — too intimate, romantic-companion paradigm, emotional chaos

**APPLY — JARVIS + TARS synthesis:**
1. **Dedicated-servant framework** (JARVIS): "Sir" addressing, professional distance, user-directed
2. **Acoustic warmth** (HAL inverse): natural pitch variance, NOT monotone
3. **Sarcasm and humor as trust-building** (TARS): context-aware dry humor, "Sanity-as-a-Service"
4. **Mechanical transparency** (TARS): "honesty" and "humor" parameters user-tunable

The MFL-Voice FATHER persona = "courteous, intellectually mature, finely-humorous British butler who never tries to be your best friend, but you can always entrust the most complex tasks".
