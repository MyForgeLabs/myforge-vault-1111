# 11.11 session protokoll

A vault saját **work-unit** szervezési rendszere: minden session egy `08-Sessions/YYYY-MM-DD-<slug>.md` fájl, dedikált bash-parancs-családdal. **Több session lehet egyszerre nyitva**, fókusz váltogatható.

## Parancsok

| Parancs | Mit csinál |
|---------|-----------|
| `11.11` | Health check — vault git, symlinkek, skillek, szolgáltatások, **összes nyitott** session, utolsó projektek |
| `11.11start "név"` | Új session fájl + focus állítás (új session lesz a focused) |
| `11.11ls` / `11.11list` | Nyitott session-ök + utolsó 5 zárt |
| `11.11focus <slug>` | Focus váltása másik nyitott session-re |
| `11.11note "..."` | Timestamped jegyzet a **focused** session `## Events`-ébe |
| `11.11note @<slug> "..."` | Jegyzet konkrét session-be (substring match) |
| `11.11stop` | Agent: review chat → Summary + Learnings + Next; Script: commit + push + close (focused) |
| `11.11stop <slug>` | Konkrét session zárása |

## Session-fájl séma

```yaml
---
name: <ember-olvasható cím>
project: <slug — egyezik 02-Projects/<slug>.md-vel ha van>
status: open | closed
started: 2026-04-23T18:37+00:00
ended:                                 # ISO ha closed
agent: claude | codex | gemini
tags: ["#type/session", "#project/<slug>"]
---
## Cél
## Events
## Summary
## Learnings → memória
## Next session
```

## Stop-kor az agent felelőssége

A bash-script **csak a mechanikus részt** csinálja (ended timestamp, status closed, commit, push). A **Summary / Learnings / Next** szekciókat az **agent** tölti ki a chat-history alapján:

1. Olvassa át a chat-et + a `## Events` jegyzeteket
2. Ír egy **Summary**-t (1-3 bekezdés)
3. Kiemeli a **Learnings**-et (bullet-ek)
4. A Learnings releváns bullet-jeit átpropagálja:
   - Projekt-specifikus tanulság → `02-Projects/<slug>.md`
   - User-preferencia → `05-Memory/User.md`
   - Infra-tudás → `05-Memory/Infrastructure.md`
   - Architektúra-döntés → új `07-Decisions/YYYY-MM-DD <téma>.md`
5. **Next session** — mivel kell folytatni, blockerek

## Hol fut

- **Scripts:** `/usr/local/bin/11.11*` + `/11.11*` symlinkek (mindhárom gépen: dev, prod, Mac)
- **Slash command:** `/11.11` Claude Code-ban is hívható
- **Agent env var:** `AGENT=claude|codex|gemini` a commit üzenetbe kerül (`stop[claude]: ...`)

## Karpathy-szempontból

A 11.11 session-fájlja az **agent-munkavault** rétegben él (Karpathy minta szerint): rendezetlen, spekulatív, ami éppen történik. A `## Learnings` **crystallization** lépést indít: amit megtanultunk, az átkerül a strukturált rétegekbe (Memory/, Decisions/, wiki/).

## Üres / duplikátum session zárása (housekeeping)

Gyakori pattern: véletlenül 2× megnyitott projekt-slug, vagy 0-jegyzetes session ami napokig nyitva marad. A standard `11.11stop` panaszkodik hogy a Summary üres — ilyenkor:

1. **Force-close:** `AGENT=claude /usr/local/bin/11.11stop <slug> --force`
2. **Summary szekció minimum:** "Üresen nyitva maradt session (duplikátum) — a tényleges munka a `YYYY-MM-DD-<slug>.md` session-ben történt. Force-close YYYY-MM-DD-én."
3. **Learnings:** `_(nincs — üres session)_`
4. **Next session:** `_(n/a)_`
5. **Propagation log:** `- YYYY-MM-DDTHH:MM — nincs propagálandó tartalom (üres session, force-close)`
6. **`.last-stop-prompt` törlése:** `rm /root/obsidian-vault/.last-stop-prompt` (a crystallization-agent nem fut le rajta)

Ezzel a session zárva van, a vault tiszta marad, és a Sessions/ mappa nem tele üres boilerplate-tel.
