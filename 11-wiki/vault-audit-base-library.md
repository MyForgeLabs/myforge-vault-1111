# `vault_audit_base` â€” shared scaffolding for audit-CLIs

A thin Python facade over `vault_atomic` that consolidates the frontmatter + body builders shared by every `vault-*-audit` / `vault-*-watch` CLI.

## Why it exists

The vault has 60+ `vault-*` CLIs that write to `06-Audits/`. Of those, ~7-10 are pure-audit shape (frontmatter + summary + tables + sections), and each one was re-implementing:

- the YAML frontmatter (`---\nname:\ntype:\ncreated:\nupdated:\ntags:\n---`)
- the `# Title` + `> Auto-generated by ... at <ts>` callout
- markdown-table rendering (pipe + separator + per-row formatter)
- timestamp helpers (`datetime.now(timezone.utc).strftime(...)`)
- audit-file path templating (`06-Audits/<slug>.md` and `<slug>-YYYY-MM-DD.json`)

Per-CLI this is ~50-80 lines of boilerplate. Frontmatter drift was creeping in: some CLIs had `generated_by:`, some didn't; some quoted tags, some used bare arrays; some used `created: <iso-date>`, some used `created: <iso-datetime>+0000`.

## Public API

```python
from vault_audit_base import (
    AuditDoc, markdown_table,
    utc_now_iso, today_iso,
    audit_path, vault_root,
)
```

### `AuditDoc(name, *, type='audit', tags=(), related=(), generated_by=None, extra_fm=None)`

Builder. Always emits `name / type / created / updated`. Appends `tags`, `related`, `generated_by`, then anything in `extra_fm`.

Body builders are chainable:

```python
doc = AuditDoc(name="ADR Status Tracker",
               tags=["#type/audit", "#audit/adr"],
               generated_by="vault-adr-aging-watch")
doc.h1("ADR Status Tracker")
doc.note(f"Total: **{total}**, Aging: **{aging}**")
doc.h2("Status distribution")
doc.table(["Status", "Count:"], status_rows)
doc.write_md(audit_path("adr-status"))
```

Methods: `.line()`, `.lines(*ss)`, `.blank()`, `.h1()/.h2()/.h3()`, `.note(text, callout="info"|"warning"|"todo"|"success")`, `.auto_note()`, `.table(headers, rows)`, `.render()`, `.write_md(path)`, `.write_json_sidecar(path, obj)`.

**Right-aligned columns:** end the header label with a colon (`"Count:"`). The colon is stripped from the rendered output and the column gets `---:`.

### `audit_path(slug, *, ext='md', date=False)`

`audit_path("adr-status")` â†’ `06-Audits/adr-status.md`
`audit_path("conflicts", date=True)` â†’ `06-Audits/conflicts-2026-05-25.md`
`audit_path("data", ext="json", date="2026-05-19")` â†’ `06-Audits/data-2026-05-19.json`

Reads `$VAULT_ROOT` (default `/root/obsidian-vault`), so a worktree-running test never clobbers the live vault.

### `utc_now_iso()` / `today_iso()`

`"2026-05-25T20:55:12Z"` and `"2026-05-25"`. Use the second-precision UTC form for `generated_at:` style fields, the date form for `created:` / `updated:`.

### `markdown_table(headers, rows)` (standalone)

Same renderer the `AuditDoc.table()` method uses, exposed for CLIs that build the markdown buffer manually but want consistent table formatting.

## What it deliberately does NOT do

- **No enforced shape** â€” CLIs with custom structure (live dashboards, multi-callout reports) keep their own render and borrow only `markdown_table()` + `audit_path()`.
- **No DB / Memgraph coupling** â€” pure string + Path work. Anything that touches Memgraph or KO-DB stays in the calling CLI.
- **No CLI argparse helper** â€” too much variance per CLI. Each owns its own arg-set.

## When to use it

Use when your CLI:

1. Writes a `06-Audits/<slug>.md` document, AND
2. Has a YAML frontmatter at the top, AND
3. Has at least one section with a table or summary callout.

Skip it when your CLI:

- Writes JSONL audit logs (use `atomic_append_jsonl` from `vault_atomic`)
- Writes a non-audit artifact (changelog, schema dump, etc.)
- Has a one-shot render that's <20 lines (just call `atomic_write` directly)
